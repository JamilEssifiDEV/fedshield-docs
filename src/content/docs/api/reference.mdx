## Grab the spec

The OpenAPI 3.1 document is hosted on this site so any OpenAPI-compatible tool can consume it:

```sh
curl https://fedshield-docs.pages.dev/openapi.yaml > fedshield.openapi.yaml
```

It mirrors the source of truth in the federation-core repo. Updates land in the repo first; this site rebuilds from it on every deploy.

## Useful tooling

- **[`openapi-generator`](https://openapi-generator.tech/)** — generate idiomatic clients for ~50 languages.
- **[`oapi-codegen`](https://github.com/oapi-codegen/oapi-codegen)** — Go-native code generation.
- **[Bruno](https://www.usebruno.com/)** or **[Insomnia](https://insomnia.rest/)** — open the spec, get a ready-made request collection.
- **[Spectral](https://stoplight.io/open-source/spectral)** — lint the spec yourself. The repo's CI runs Spectral on every push, so you can run the same ruleset locally.
